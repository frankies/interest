# 代码审查

对Java代码进行全面的Code Review，输出详细的评审报告。

## 使用方式
在聊天中输入：`/review-code [文件路径]`

## 审查维度

### 1. 代码规范检查
- 命名规范（类、方法、变量、常量）
- 注释规范（类注释、方法注释）
- 代码风格（阿里巴巴手册、Google Style）

### 2. 代码质量检查
- 单一职责原则
- 圈复杂度（< 10）
- 方法长度（< 80行）
- 重复代码检测

### 3. 性能问题检查
- 内存泄漏风险
- 不必要的对象创建
- 循环中的耗时操作
- 缓存使用合理性

### 4. 并发安全检查
- 线程安全问题
- 锁的使用
- 原子性保证
- 死锁风险

### 5. 安全隐患检查
- SQL注入风险
- XSS漏洞
- 敏感信息泄露
- 权限校验缺失

### 6. 可维护性检查
- 代码可读性
- 模块耦合度
- 异常处理完整性
- 日志记录合理性

## 输出格式
- 关键问题（必须修复）
- 改进建议（建议修复）
- 代码亮点（做得好）
- 具体修改示例
- SonarQube规则映射
- 总体评分

## 示例
```
/review-code src/main/java/com/example/service/UserService.java
```

对用户服务进行全面代码审查。
