# 性能优化

对现有代码进行性能优化，提升系统吞吐量和响应速度。

## 使用方式
在聊天中输入：`/optimize-performance [文件路径或代码片段]`

## 优化维度

### 1. 缓存优化
- 添加多级缓存（本地缓存 + Redis）
- 缓存预热和更新策略
- 缓存穿透/击穿/雪崩防护

### 2. 数据库优化
- 索引设计和优化
- 查询语句优化（避免N+1问题）
- 批量操作替代循环查询
- 分页查询优化

### 3. 并发优化
- 线程池配置优化
- 异步处理（@Async）
- 并行流处理
- 锁优化（减小锁粒度）

### 4. 代码优化
- 避免不必要的对象创建
- 使用StringBuilder替代字符串拼接
- 集合初始化容量设置
- 懒加载和延迟初始化

## 示例
```
/optimize-performance src/main/java/com/example/service/OrderService.java
```

分析并优化订单服务的性能问题。
